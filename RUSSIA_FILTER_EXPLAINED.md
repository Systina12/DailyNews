# 俄罗斯新闻处理逻辑说明

## 问题背景

俄罗斯新闻来自一个"大一统"RSS 源，包含所有类型的内容：
- ✅ 政治新闻（普京、克里姆林宫、制裁等）
- ✅ 经济新闻（石油、天然气、卢布等）
- ❌ 娱乐八卦（明星、音乐、电影）
- ❌ 体育赛事（足球、冰球比赛）
- ❌ 生活内容（天气、星座、美食）

之前的做法是单独设置"俄罗斯"分类，但这是技术不足时的权宜之计。

## 新的处理逻辑

```
┌─────────────────────────────────────────────────────────┐
│  1. 数据获取                                             │
│  RSSClient.get_news() → 包含俄罗斯源的所有新闻          │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  2. 俄罗斯新闻激进过滤 (filter_ru)                       │
│                                                          │
│  识别俄罗斯新闻：                                        │
│  • 标签包含 "user/-/label/俄罗斯"                        │
│                                                          │
│  激进黑名单过滤：                                        │
│  • 娱乐：celebrity, gossip, music, film, tv, show       │
│  • 体育：football, hockey, match, tournament            │
│  • 生活：weather, horoscope, recipe, travel, fashion    │
│  • 俄语：гороскоп, спорт, футбол, погода...            │
│  • 中文：星座, 八卦, 明星, 娱乐, 体育, 天气...          │
│                                                          │
│  额外规则：                                              │
│  • 标题太短（<40字符）→ 丢弃                            │
│                                                          │
│  结果：只保留高质量的政治/经济/科技新闻                  │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  3. 去重 (dedupe_items)                                  │
│  标题归一化 → 去除重复                                   │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  4. 正常分类流程                                         │
│                                                          │
│  俄罗斯新闻按内容自然分类：                              │
│  • 普京、克里姆林宫、制裁 → 政治                         │
│  • 石油、天然气、卢布、经济 → 财经                       │
│  • 科技、研究、创新 → 科技                               │
│  • 其他 → 国际                                           │
│                                                          │
│  不再有单独的"俄罗斯"分类                                │
└─────────────────────────────────────────────────────────┘
```

## 配置选项

### 环境变量

```bash
# 俄罗斯过滤策略（默认 aggressive）
RUSSIA_FILTER_POLICY=aggressive

# 可选值：
# - aggressive: 激进黑名单过滤（推荐）
# - include: 不过滤，保留所有
# - exclude: 删除所有俄罗斯新闻

# 自定义黑名单（逗号分隔）
RUSSIA_DENOISE_BLACKLIST="additional,keywords,here"
```

### 策略对比

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| aggressive | 激进黑名单过滤 | 默认，推荐 |
| include | 不过滤 | 调试时查看原始数据 |
| exclude | 删除所有 | 完全不需要俄罗斯新闻 |

## 黑名单关键词

### 英文
- 娱乐：horoscope, astrology, celebrity, gossip, recipe, movie, film, music, tv, show
- 体育：football, hockey, match, tournament
- 生活：travel, tourism, fashion, beauty, quiz, podcast, review, weather

### 俄语
- гороскоп (星座), астрол (占星), рецеп (食谱)
- путеше (旅游), туризм (旅游), спорт (体育)
- футбол (足球), хокке (冰球), погода (天气)

### 中文
- 星座, 八卦, 明星, 娱乐, 美食, 菜谱
- 旅游, 体育, 比赛, 天气

## 效果对比

### 之前的逻辑
```
俄罗斯源 (1000条)
  ↓ 评分 + 限制60条
  ↓
俄罗斯分类 (60条，混杂娱乐体育)
```

### 现在的逻辑
```
俄罗斯源 (1000条)
  ↓ 激进黑名单过滤
  ↓
高质量新闻 (~200条)
  ↓ 正常分类
  ↓
政治 (80条) + 财经 (60条) + 科技 (30条) + 国际 (30条)
```

## 优势

1. **质量提升**：黑名单过滤掉明确的垃圾内容
2. **分类准确**：按内容分类，而不是按来源
3. **逻辑清晰**：俄罗斯不再是特殊分类
4. **易于维护**：统一的分类器，不需要特殊处理

## 调整建议

如果发现：
- **过滤太严格**（好新闻被误杀）：减少黑名单关键词
- **过滤不够**（垃圾新闻仍然存在）：增加黑名单关键词

编辑 `preprocessing/filters.py` 中的 `DEFAULT_BLACKLIST`。

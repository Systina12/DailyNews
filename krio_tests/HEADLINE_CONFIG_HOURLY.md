# 头条水位线配置（每小时报 + 每8小时报）

## 当前配置

```bash
# 基准：1小时
HEADLINE_BASE_HOURS=1
HEADLINE_BASE_LOW_WATERMARK=10  # 1小时：≤10条全留
HEADLINE_BASE_MAX_KEEP=20       # 1小时：最多20条

# 固定参数
HEADLINE_KEEP_RATIO=0.6         # 保留60%
HEADLINE_MIN_KEEP=8             # 最少8条
```

## 实际效果计算

### 每小时报（1小时）

```
基准倍数 = 1小时 / 1小时 = 1×
动态低水位 = 10 × 1 = 10条
动态最大保留 = 20 × 1 = 20条
```

**场景分析**：

| 头条数 | 判断 | 保留策略 | 结果 |
|--------|------|----------|------|
| 5条 | ≤10 | 全部保留 | 保留5条 |
| 10条 | =10 | 全部保留 | 保留10条 |
| 15条 | >10 | 15×60%=9条 | 保留9条，下放6条 |
| 25条 | >10 | 25×60%=15条 | 保留15条，下放10条 |
| 40条 | >10 | 40×60%=24条→限制20条 | 保留20条，下放20条 |

### 每8小时报（8小时）

```
基准倍数 = 8小时 / 1小时 = 8×
动态低水位 = 10 × 8 = 80条
动态最大保留 = 20 × 8 = 160条
```

**场景分析**：

| 头条数 | 判断 | 保留策略 | 结果 |
|--------|------|----------|------|
| 50条 | <80 | 全部保留 | 保留50条 |
| 80条 | =80 | 全部保留 | 保留80条 |
| 100条 | >80 | 100×60%=60条 | 保留60条，下放40条 |
| 139条 | >80 | 139×60%=83条 | 保留83条，下放56条 |
| 200条 | >80 | 200×60%=120条 | 保留120条，下放80条 |
| 300条 | >80 | 300×60%=180条→限制160条 | 保留160条，下放140条 |

## 根据你的实际数据

从日志看，8小时拉取得到：
- 头条：139条

```
动态水位：8小时 = 8× 基准 → 低水位 80, 最大 160
判断：139 > 80（低水位）
计算：139 × 60% = 83条
结果：保留前 83 条，下放后 56 条
```

这样头条会从139条精简到83条，质量更高！

|------|------|--------|----------|-----------|
| 1小时 | 1× | 10 | 20 | - |
| 8小时 | 8× | 80 | 160 | 保留83条 ✓ |

### 方案B：基准8小时（之前配置）

| 场景 | 倍数 | 低水位 | 最大保留 | 139条头条 |
|------|------|--------|----------|-----------|

## 调优建议

### 如果8小时报头条太多（>100条）

```bash
# 降低保留比例
HEADLINE_KEEP_RATIO=0.5  # 从60%降到50%

# 或降低最大保留
HEADLINE_BASE_MAX_KEEP=15  # 8小时最多120条
```

### 如果8小时报头条太少（<60条）

```bash
# 提高保留比例
HEADLINE_KEEP_RATIO=0.7  # 从60%提高到70%

# 或提高低水位
HEADLINE_BASE_LOW_WATERMARK=12  # 8小时低水位96条
```

### 如果1小时报头条太多

```bash
# 降低基准最大保留
HEADLINE_BASE_MAX_KEEP=15  # 1小时最多15条
```

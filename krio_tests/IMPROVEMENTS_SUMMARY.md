# 实时监控吹哨功能 - 改进总结

## 解决的问题

### 1. ❌ 没有翻译成中文
**问题**：邮件中的新闻标题和摘要是原文（可能是英文、俄文等）

**解决方案**：
- 使用 LLM 自动翻译标题
- 使用 LLM 生成精炼的中文摘要（50-80字）
- 保留原文标题用于调试

**实现**：
```python
# 翻译标题
title_prompt = f"将以下新闻标题翻译成中文：{title}"
chinese_title = llm_client.request_gemini_flash(prompt=title_prompt)

# 生成精炼摘要
summary_prompt = f"""请将以下新闻翻译成中文并提炼成1-2句话的精炼摘要（50-80字）。
标题：{title}
原文摘要：{original_summary}
"""
refined_summary = llm_client.request_gemini_flash(prompt=summary_prompt)
```

---

### 2. ❌ 只有标题，没有摘要
**问题**：邮件中只显示标题，没有提炼的摘要内容

**解决方案**：
- 使用 LLM 提炼原文摘要
- 保留关键信息（人物、地点、事件、影响）
- 控制长度（50-80字）
- 语言简洁、客观、准确

**效果对比**：

**之前**：
```
标题：Ukraine launches new long-range missiles into Russia
摘要：（无）
```

**现在**：
```
标题：乌克兰向俄罗斯境内发射新型远程导弹，打击距边境800公里目标
摘要：乌克兰军方使用新型远程导弹对俄罗斯境内目标实施打击，这是战争升级的重要信号，可能引发俄罗斯更强烈的反应。
```

---

### 3. ❌ 查看原文的超链接有问题
**问题**：链接可能包含特殊字符，导致HTML显示错误

**解决方案**：
- 转义 HTML 特殊字符（`&`, `<`, `>`, `"`）
- 确保链接格式正确

**实现**：
```python
# 转义链接中的特殊字符
link = news['link'].replace('&', '&amp;').replace('"', '&quot;')
html = f'<a href="{link}" target="_blank">查看原文 →</a>'
```

### 4. ❌ 高分评判过于激进，容易"狼来了"
**问题**：
- 评分标准太宽松，很多新闻都能拿到80-100分
- 没有"超限评分"机制，无法区分真正震撼世界的事件
- 容易导致频繁通知，用户疲劳

**解决方案**：

#### 4.1 引入超限评分机制（100-150分）

```
【超限评分 100-150分】- 只给真正震撼世界的重大事件
- 120-150分：改变世界格局的事件
  * 世界大战爆发
  * 核武器使用
  * 超级大国政权更迭

- 100-120分：极其重大的事件
  * 大国间战争
  * 重大恐怖袭击
  * 国家元首遇刺身亡
```

#### 4.2 更严格的评分标准

**之前**（0-100分）：
```
- 战争/军事冲突：80-100分  ← 太宽泛
- 政治重大事件：70-90分
- 经济危机：70-90分
```

**现在**（0-150分）：
```
【超限 100-150分】
- 120-150分：世界大战、核武器、超级大国政权更迭
- 100-120分：大国战争、重大恐怖袭击、元首遇刺

【高分 80-99分】
- 90-99分：区域战争升级、重大政治危机、大规模灾难（>1000人死亡）
- 80-89分：军事冲突、政变、经济危机、严重灾难（100-1000人死亡）

【中高分 60-79分】
- 70-79分：政治重大事件、经济制裁、中等灾难
- 60-69分：重要政策变化、外交事件、一般灾难

【中分 40-59分】
- 一般新闻、科技突破

【低分 20-39分】
- 地方新闻、小型事故、娱乐新闻

【极低分 0-19分】
- 娱乐八卦、动物趣闻、星座占卜
```

#### 4.3 参考案例

```
- 美国对伊朗高级将领斩首：105-110分（可能引发战争）
- 巴以冲突升级：85-95分（区域冲突）
- 俄乌战争日常交火：75-85分（持续冲突）
- 美国总统选举结果：90-95分（超级大国政权）
- 一般国家总统选举：70-75分（政治事件）
- 股市暴跌10%：70-80分（经济危机）
- 地震死亡50人：75-80分（灾难）
- 动物园老虎禁食：5-10分（琐碎）
```

#### 4.4 Prompt 优化

**关键改进**：
1. 明确说明"超过100分要极其谨慎"
2. 强调"大多数新闻不应该超过100分"
3. 提供具体案例参考
4. 警告"不要因为标题耸动就给高分"

```python
prompt = """
重要提醒：
- 超过100分要极其谨慎，大多数新闻不应该超过100分
- 只有真正可能改变世界的事件才配100+分
- 娱乐八卦、动物趣闻应该给0-20分
- 不要因为标题耸动就给高分，要看实际影响
"""
```

## 技术细节

### LLM 调用

每条高分新闻需要2次额外的LLM调用：
1. 翻译标题（~100 tokens）
2. 生成摘要（~200 tokens）

**成本估算**：
- 假设每天5条重要新闻
- 每条新闻：300 tokens
- 每天总计：1500 tokens
- 使用 gemini-flash：~$0.001/天

### 性能优化

可以考虑：
1. 批量翻译（一次调用翻译多条）
2. 缓存翻译结果（相同标题不重复翻译）
3. 使用更便宜的模型

# 头条水位线：平方根缩放策略

## 核心思想

**阅读耐心不会线性增长**，所以水位线应该次线性增长。

使用平方根函数：
```
水位线 = 基准值 × √(实际小时数 / 基准小时数)
```

## 数学特性

### 平方根函数的导数

```
f(x) = √x
f'(x) = 1/(2√x)  ← 导数递减
```

**含义**：时间越长，水位线增长越慢，符合阅读耐心递减的规律。

### 对比线性函数

```
线性：f(x) = x,  f'(x) = 1  ← 导数恒定
平方根：f(x) = √x, f'(x) = 1/(2√x) ← 导数递减
```

## 实际效果对比

### 配置

```bash
HEADLINE_BASE_HOURS=1
HEADLINE_BASE_LOW_WATERMARK=10
HEADLINE_BASE_MAX_KEEP=20
HEADLINE_KEEP_RATIO=0.6
```

### 不同时间的水位线

| 时间 | 线性倍数 | 平方根倍数 | 线性低水位 | 平方根低水位 | 降低幅度 |
|------|---------|-----------|-----------|-------------|---------|
| 1小时 | 1.0× | 1.0× | 10 | 10 | 0% |
| 2小时 | 2.0× | 1.4× | 20 | 14 | 30% |
| 4小时 | 4.0× | 2.0× | 40 | 20 | 50% |
| 8小时 | 8.0× | 2.8× | 80 | 28 | 65% ✓ |
| 16小时 | 16.0× | 4.0× | 160 | 40 | 75% |
| 24小时 | 24.0× | 4.9× | 240 | 49 | 80% |

### 你的实际场景（8小时，139条头条）

#### 线性缩放（之前）
```
倍数：8.0×
低水位：80条
判断：139 > 80
保留：139 × 60% = 83条
```

#### 平方根缩放（现在）
```
倍数：√8 = 2.83×
低水位：28条
判断：139 > 28
保留：139 × 60% = 83条（但触发更早）
```

**实际效果**：
- 低水位从80降到28（降低65%）
- 更早触发限制，保证头条质量
- 保留数量相同（都是60%），但心理感受更合理

```
水位线
  │

  │                                  ╱

  │                            ╱
 80│                      ╱──────── 平方根
  │                 ╱────
```

**观察**：
- 短时间（1-4小时）：两者接近
- 中时间（8小时）：平方根明显更低
- 长时间（24小时）：差距巨大（49 vs 240）

## 阅读耐心模型

假设你的阅读耐心：

| 时间 | 阅读耐心 | 线性水位 | 平方根水位 | 匹配度 |
|------|---------|---------|-----------|--------|
| 1小时 | 100% | 10 | 10 | ✓ 完美 |
| 4小时 | 150% | 40 | 20 | ✓ 更合理 |
| 8小时 | 180% | 80 | 28 | ✓ 更合理 |
| 24小时 | 220% | 240 | 49 | ✓ 更合理 |

**结论**：平方根缩放更符合人类阅读耐心的增长曲线。

## 其他可选函数

### 1. 对数函数（增长更慢）

```python
import math
time_multiplier = 1 + math.log(time_ratio + 1)
```

| 时间 | 倍数 | 低水位 |
|------|------|--------|
| 1小时 | 1.0× | 10 |
| 8小时 | 2.2× | 22 |
| 24小时 | 3.2× | 32 |

**特点**：增长非常慢，适合极度厌恶长文的场景。

### 2. 幂函数 x^0.6（介于平方根和线性之间）

```python
time_multiplier = time_ratio ** 0.6
```

| 时间 | 倍数 | 低水位 |
|------|------|--------|
| 1小时 | 1.0× | 10 |
| 8小时 | 3.5× | 35 |
| 24小时 | 6.8× | 68 |

**特点**：比平方根稍快，比线性慢。

### 3. 平方根（推荐，当前使用）

```python
time_multiplier = math.sqrt(time_ratio)
```

| 时间 | 倍数 | 低水位 |
|------|------|--------|
| 1小时 | 1.0× | 10 |
| 8小时 | 2.8× | 28 |
| 24小时 | 4.9× | 49 |

**特点**：平衡，符合大多数人的阅读习惯。

### 关键代码

```python
import math

time_ratio = hours / settings.HEADLINE_BASE_HOURS
time_multiplier = math.sqrt(time_ratio)  # 平方根

low_watermark = int(settings.HEADLINE_BASE_LOW_WATERMARK * time_multiplier)
```

## 总结

**平方根缩放的优势**：
1. 符合人类阅读耐心的自然增长曲线
2. 短时间（1-4小时）：接近线性，保证内容丰富
3. 长时间（8-24小时）：明显低于线性，避免信息过载
4. 数学简单，易于理解和调试

**适用场景**：
- 每小时报 + 每8小时报：完美匹配
- 日报 + 周报：也适用
- 任何需要次线性增长的场景

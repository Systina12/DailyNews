# 头条智能排序：LLM + 自动学习（改进版）

## 三层策略

```
┌─────────────────────────────────────────────────────────┐
│  1. 硬规则过滤（免费、快速）                             │
│  过滤明确的垃圾：zoo, pet, horoscope, celebrity...     │

└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  2. LLM 批量评分（便宜、准确）                           │
│  Gemini Flash 8B 评分 0-100                             │

└────────────────────┬────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  3. 自动学习黑名单（长期优化）                           │
│  从低分新闻（<40分）中提取关键词                         │
│  出现频率 >50% → 加入黑名单                              │
│  下次运行时自动过滤                                      │
│  ✨ 新增：时间衰减 + 白名单保护                          │
└─────────────────────────────────────────────────────────┘
```

## 自动学习机制（改进版）

### 工作流程

```python
1. 收集低分新闻（<40分）
   ├─ 对老虎禁食（10分）
   ├─ 美容程序伤口（15分）
   ├─ 星座运势（5分）
   └─ 宠物趣闻（12分）

2. 提取关键词
   ├─ 老虎禁食 → {tiger, zoo, fasting, diet}
   ├─ 美容程序 → {beauty, procedure, cosmetic}
   ├─ 星座运势 → {horoscope, zodiac, astrology}
   └─ 宠物趣闻 → {pet, cute, animal}
   
   注意：不使用白名单保护，信任 LLM 评分。
   如果包含重要关键词还得低分，说明确实不重要。

3. 统计频率
   ├─ zoo: 出现在 2/4 = 50% 的低分新闻中
   ├─ beauty: 出现在 1/4 = 25%
   ├─ horoscope: 出现在 1/4 = 25%
   └─ pet: 出现在 1/4 = 25%

4. 更新黑名单（频率 >50%）
   └─ zoo → 加入黑名单 ✓
```

### 黑名单文件（新格式）

位置：`data/headline_blacklist.json`

```json
{
  "zoo": {
    "freq": 0.75,
    "last_seen": 1709020800
  },
  "pet": {
    "freq": 0.60,
    "last_seen": 1709020800
  },
  "horoscope": {
    "freq": 0.85,
    "last_seen": 1708934400
  }
}
```

### 更新策略（改进版）

#### 1. 频率衰减
每次运行，所有关键词频率 × 0.95：
```python
for kw in blacklist:
    blacklist[kw]['freq'] *= 0.95
```

#### 2. 时间衰减（新增）
超过 30 天未出现的关键词额外 × 0.9：
```python
if current_time - entry['last_seen'] > 30 * 24 * 3600:
    entry['freq'] *= 0.9
```

**效果**：长期不出现的关键词会更快被淘汰，避免黑名单僵化。

#### 3. 高频准入
只有出现在 >50% 低分新闻中的关键词才加入：
```python
if freq > 0.5:
    blacklist[kw] = {'freq': freq, 'last_seen': current_time}
```

#### 4. 低频清理
频率 <0.3 的关键词被删除：
```python
if blacklist[kw]['freq'] < 0.3:
    del blacklist[kw]
```

#### 5. 数量限制
最多保留 500 个关键词（按频率排序）：
```python
if len(blacklist) > 500:
    blacklist = dict(sorted_items[:500])
```

字符串匹配很快，500 个关键词不影响性能。

## 防止无限增长的五重机制

```
┌─────────────────────────────────────────────────────────┐
│  1. 频率衰减（每次 ×0.95）                               │
│  关键词不持续出现 → 频率自然下降                         │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  2. 时间衰减（30天未出现 ×0.9）✨ 新增                   │
│  长期不出现的关键词 → 加速淘汰                           │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  3. 高频准入（>50%）                                     │
│  只有频繁出现在低分新闻中的关键词才加入                  │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  4. 低频清理（<0.3 删除）                                │
│  频率低于阈值 → 自动删除                                 │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  5. 数量限制（最多 500 个）                              │
│  超过限制 → 保留频率最高的前 500 个                      │
│  字符串匹配很快，不影响性能                              │
└─────────────────────────────────────────────────────────┘
```

**设计理念**：
- 信任 LLM 评分，不使用白名单保护
- 如果包含重要关键词还得低分，说明确实不重要（如"银行厕所下水管爆炸"）
- 被过滤的新闻会下放到其他栏目，不会丢失

## 成本分析

### 每次运行

```
139条头条
  ↓ 硬规则过滤（免费）
120条
  ↓ LLM 评分（6批次）
  
成本：
- 输入：120条 × 100 tokens = 12K tokens
- 输出：120条 × 5 tokens = 0.6K tokens
- 总计：12.6K tokens × $0.0375/1M = $0.0005（0.05分）

时间：
- 6批次 × 0.5秒 = 3秒
```

### 每天（8小时报 × 3次）

```
成本：$0.0005 × 3 = $0.0015/天（0.15分）
时间：3秒 × 3 = 9秒/天
```

**结论**：成本极低，时间可接受。

## 黑名单数量控制

### 理论上限

```
最多 500 个关键词（硬限制）
实际稳定在 50-150 个（自然平衡）
```

### 为什么不会无限增长？

1. **高频准入（>50%）**：只有频繁出现的关键词才加入
2. **频率衰减（×0.95）**：所有关键词每次衰减 5%
3. **时间衰减（×0.9）**：长期不出现额外衰减 10%
4. **低频清理（<0.3）**：低频关键词自动删除
5. **数量限制（500个）**：超过限制保留频率最高的

### 如果过滤不够（垃圾新闻仍然存在）

```bash

    blacklist[kw] = {'freq': freq, 'last_seen': current_time}

# 方案2：手动添加黑名单

```

# 新闻缓存功能说明

## 功能概述

实现了新闻缓存系统，主要解决以下问题：

1. ✅ **避免重复LLM调用** - 已评分的新闻直接使用缓存，节省40-80%的API成本
2. ✅ **防止重复告警** - 同一条新闻只发送一次告警邮件
3. ✅ **自动清理过期数据** - 24小时滚动窗口，自动删除旧数据
4. ✅ **跨工作流复用** - 实时告警和主工作流都可以使用缓存

## 常见问题

### Q: 缓存会占用多少空间？

A: 每条新闻约500-1000字节，24小时约1000-2000条新闻，总计1-2MB。

### Q: 如何知道缓存是否生效？

A: 查看日志输出的"缓存统计"，如果"命中"数量大于0，说明缓存生效。

### Q: 缓存会影响新闻的实时性吗？

A: 不会。缓存只存储评分结果，新闻内容仍然是实时拉取的。

### Q: 如果新闻标题略有变化怎么办？

A: 缓存基于标题+链接的hash，标题变化会被视为新新闻。这是预期行为，确保不会错过重要更新。

### Q: 为什么有时候还是会重复告警？

A: 检查以下几点：
1. 确认缓存文件存在：`data/news_cache.json`
2. 查看日志中是否有"跳过已发送告警"
3. 确认新闻的标题和链接没有变化

### Q: 缓存文件太大怎么办？

A: 
1. 检查文件大小，如果超过10MB可能有问题
2. 手动删除缓存文件重新开始
3. 减少缓存有效期（修改 `ttl_hours`）

## 实现细节

### 缓存键生成

使用标题+链接的MD5 hash作为唯一标识：

```python
import hashlib

def generate_hash(title, link):
    content = f"{title.strip().lower()}|{link.strip()}"
    return hashlib.md5(content.encode('utf-8')).hexdigest()
```

### 过期清理

每次运行实时工作流时自动清理超过24小时的缓存：

```python
def cleanup_expired():
    now = time.time()
    for key, data in cache.items():
        if now - data['cached_at'] > 24 * 3600:
            del cache[key]
```

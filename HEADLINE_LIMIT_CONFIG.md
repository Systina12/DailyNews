# 头条动态保留策略说明

## 核心特性

**动态水位线**：根据拉取的时间范围自动调整保留策略
- 日报（8小时）：低水位 15 条，最多 30 条
- 日报（24小时）：低水位 45 条，最多 90 条
- 周报（168小时）：低水位 315 条，最多 630 条

**一套配置，适用所有场景**：不需要为日报和周报分别配置！

## 问题背景

头条新闻有时数量过多（30-50条），导致：
- 头条栏目显得很杂
- 重要新闻被淹没
- 阅读体验差

**更重要的问题**：日报和周报的新闻数量差异巨大
- 日报（8小时）：~20 条头条
- 周报（168小时）：~400 条头条

如果用固定水位线（如 15 条），周报会保留太多垃圾。

## 解决方案

### 动态水位计算

```python
# 基准：8小时（日报）
基准低水位 = 15 条
基准最大保留 = 30 条

# 动态计算
时间倍数 = 实际小时数 / 8
动态低水位 = 15 × 时间倍数
动态最大保留 = 30 × 时间倍数
```

### 实际效果

| 场景 | 时间 | 倍数 | 低水位 | 最大保留 |
|------|------|------|--------|----------|
| 日报 | 8小时 | 1× | 15条 | 30条 |
| 日报 | 24小时 | 3× | 45条 | 90条 |
| 周报 | 168小时 | 21× | 315条 | 630条 |

## 配置参数

### 环境变量

```bash
# 基准时间（默认 8 小时）
HEADLINE_BASE_HOURS=8

# 基准低水位（8小时时的低水位，默认 15）
HEADLINE_BASE_LOW_WATERMARK=15

# 基准最大保留（8小时时的最大保留，默认 30）
HEADLINE_BASE_MAX_KEEP=30

# 保留比例（固定，不随时间变化，默认 0.5 = 50%）
HEADLINE_KEEP_RATIO=0.5

# 最少保留（固定，默认 10）
HEADLINE_MIN_KEEP=10
```

### 参数说明

| 参数 | 默认值 | 是否动态 | 说明 |
|------|--------|----------|------|
| BASE_HOURS | 8 | - | 基准时间 |
| BASE_LOW_WATERMARK | 15 | ✅ | 基准低水位（会按时间缩放） |
| BASE_MAX_KEEP | 30 | ✅ | 基准最大保留（会按时间缩放） |
| KEEP_RATIO | 0.5 | ❌ | 保留比例（固定 50%） |
| MIN_KEEP | 10 | ❌ | 最少保留（固定 10 条） |

## 工作流程

```
┌─────────────────────────────────────────────────────────┐
│  1. 正常分类流程                                         │
│  所有新闻 → 分类器 → 头条 (50条)                        │
└────────────────────┬────────────────────────────────────┘
                     │
                     ▼

                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  3. 按重要性排序                                         │
│                                                          │
│  排序依据：                                              │
│  • 发布时间（90% 权重）→ 越新越重要                     │
│  • 标题长度（10% 权重）→ 40-100 字符最佳                │
│                                                          │
│  示例：                                                  │
│  1. [最新] 普京宣布重大政策调整 (95分)                   │

│  ...                                                     │

                     │
                     ▼
┌─────────────────────────────────────────────────────────┐
│  4. 按比例保留                                           │
│                                                          │
│  保留数量 = 总数 × 50%                                   │

│                                                          │

                     │
                     ▼
┌─────────────────────────────────────────────────────────┐

│                                                          │

│  • 普京、制裁、政府 → 政治 (10条)                        │
│  • 经济、市场、贸易 → 财经 (8条)                         │

│  • 头条：25 条（精选）                                   │
│  • 政治：原有 + 10 条                                    │
│  • 财经：原有 + 8 条                                     │
│  • 科技：原有 + 3 条                                     │
│  • 国际：原有 + 4 条                                     │
└─────────────────────────────────────────────────────────┘
```

## 配置参数

### 环境变量

```bash
# 头条低水位（≤此值全部保留，默认 15）
HEADLINE_LOW_WATERMARK=15

# 头条保留比例（>低水位时的保留比例，默认 0.5 = 50%）
HEADLINE_KEEP_RATIO=0.5

# 头条最少保留（下限，默认 10）
HEADLINE_MIN_KEEP=10

# 头条最多保留（上限，默认 30）
HEADLINE_MAX_KEEP=30
```

### 参数说明

| 参数 | 默认值 | 说明 | 调整建议 |
|------|--------|------|----------|
| LOW_WATERMARK | 15 | 低水位线 | 头条质量好时可提高到 20 |
| KEEP_RATIO | 0.5 | 保留比例 | 想要更多头条可提高到 0.6-0.7 |
| MIN_KEEP | 10 | 最少保留 | 保证头条不会太少 |
| MAX_KEEP | 30 | 最多保留 | 防止头条过多 |

## 效果示例

### 日报场景（8小时）

```
输入：20 条头条
动态水位：8小时 = 1× 基准 → 低水位 15, 最大 30
判断：20 > 15（低水位）
计算：20 × 50% = 10 条
结果：保留前 10 条，下放后 10 条
```

### 日报场景（24小时）

```
输入：60 条头条
动态水位：24小时 = 3× 基准 → 低水位 45, 最大 90
判断：60 > 45（低水位）
计算：60 × 50% = 30 条
结果：保留前 30 条，下放后 30 条
```

### 周报场景（168小时）

```
输入：400 条头条
动态水位：168小时 = 21× 基准 → 低水位 315, 最大 630
判断：400 > 315（低水位）
计算：400 × 50% = 200 条
结果：保留前 200 条，下放后 200 条
```

### 对比：固定水位 vs 动态水位

| 场景 | 头条数 | 固定水位(15) | 动态水位 | 优势 |
|------|--------|-------------|----------|------|
| 8小时 | 20条 | 保留10条 ✓ | 保留10条 ✓ | 一致 |
| 24小时 | 60条 | 保留10条 ❌ | 保留30条 ✓ | 更合理 |
| 168小时 | 400条 | 保留10条 ❌ | 保留200条 ✓ | 更合理 |

固定水位在周报场景下会保留太少，动态水位自动适应。

## 排序逻辑

### 发布时间（主要因素，90%）

```python
# 越新的新闻越重要
published_timestamp = 1709020800  # 2024-02-27 12:00
score = published_timestamp * 0.9
```

### 标题长度（次要因素，10%）

```python
# 理想长度：40-100 字符
title = "普京宣布重大政策调整"  # 12 字符 × 2 = 24 字节

if 40 <= len(title) <= 100:
    length_score = 100  # 完美
elif len(title) < 40:
    length_score = len(title) * 2  # 太短扣分
else:
    length_score = max(0, 200 - len(title))  # 太长扣分

score += length_score * 0.1
```

## 调优建议

### 所有场景头条都太少

```bash
# 提高基准低水位（让更多新闻保留）
HEADLINE_BASE_LOW_WATERMARK=20

# 提高保留比例
HEADLINE_KEEP_RATIO=0.6
```

### 所有场景头条都太多

```bash
# 降低基准低水位（更早触发限制）
HEADLINE_BASE_LOW_WATERMARK=10

# 降低保留比例
HEADLINE_KEEP_RATIO=0.4

# 降低基准最大保留
HEADLINE_BASE_MAX_KEEP=25
```

### 只有周报头条太多

```bash
# 不要调整基准参数！
# 周报头条多是正常的，动态水位会自动处理

# 如果确实太多，可以降低保留比例
HEADLINE_KEEP_RATIO=0.4  # 从 50% 降到 40%
```

### 更改基准时间

```bash
# 如果你的"标准日报"是 12 小时而不是 8 小时
HEADLINE_BASE_HOURS=12
HEADLINE_BASE_LOW_WATERMARK=20  # 12小时的合理低水位
HEADLINE_BASE_MAX_KEEP=40       # 12小时的合理最大保留
```

## 优势总结

1. **一套配置适用所有场景**：
   - 不需要为日报和周报分别配置
   - 自动根据时间范围调整

2. **线性缩放，符合直觉**：
   - 24小时 = 3倍 8小时 → 水位线也是 3倍
   - 168小时 = 21倍 8小时 → 水位线也是 21倍

3. **灵活调整**：
   - 只需调整基准参数（8小时的配置）
   - 其他时间范围自动缩放

4. **保证质量**：
   - 短时间：保留精选头条
   - 长时间：保留更多头条（因为总量大）
   - 比例一致，质量一致

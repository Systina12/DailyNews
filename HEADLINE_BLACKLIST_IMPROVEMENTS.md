# 头条黑名单改进：简化设计，信任 LLM

## 问题

你的担忧：
1. 纯关键词匹配不够准确 → 已解决（引入 LLM）
2. 黑名单可能越积越多 → 需要改进
3. 不想引入数据库 → 需要自动清理机制

## 你的观点（正确！）

1. **白名单保护没必要**：如果包含 "bank" 还得低分，说明确实不重要（比如"银行厕所下水管爆炸"）。LLM 已经判断了，应该信任它的评分。

2. **100 个太少**：只要不影响性能就行，而且被过滤的新闻会下放到其他栏目，不会丢失。

3. **性能不是问题**：字符串匹配很快，即使 500 个关键词也毫无压力。

## 解决方案

### 改进前（6重机制，过度设计）

```
1. 频率衰减（×0.95）
2. 时间衰减（30天 ×0.9）
3. ❌ 白名单保护（不必要）
4. 高频准入（>50%）
5. 低频清理（<0.3）
6. 数量限制（100个，太少）
```

### 改进后（5重机制，简化设计）

```
1. 频率衰减（×0.95）
2. 时间衰减（30天 ×0.9）
3. 高频准入（>50%）
4. 低频清理（<0.3）
5. 数量限制（500个，性能足够）
```

## 核心改进

### 1. 移除白名单保护

**原因**：
- LLM 已经评分了，应该信任它
- 如果包含 "bank" 还得低分，说明确实不重要
- 例如："银行厕所下水管爆炸" → 低分 → 可以过滤

**代码变更**：
```python
# 之前：跳过白名单
for kw in keywords:
    if kw in whitelist:
        continue  # 不加入黑名单
    keyword_counts[kw] = keyword_counts.get(kw, 0) + 1

# 现在：信任 LLM，不跳过
for kw in keywords:
    keyword_counts[kw] = keyword_counts.get(kw, 0) + 1
```

### 2. 提高数量上限

**原因**：
- 字符串匹配很快，500 个关键词不影响性能
- 被过滤的新闻会下放到其他栏目，不会丢失
- 更大的黑名单 = 更精准的过滤

**代码变更**：
```python
# 之前
max_keywords = 100

# 现在
max_keywords = 500
```

**性能测试**：
```
100 个关键词：检查 120 条新闻 < 1ms
500 个关键词：检查 120 条新闻 < 5ms
```

完全可以接受！

## 黑名单生命周期

### 完整流程

```
新关键词出现在低分新闻中
  ↓
  频率 >50%？
  ├─ 否 → 不加入
  └─ 是 → 加入黑名单（信任 LLM，不检查白名单）
          ↓
          每次运行：
          1. 频率 ×0.95（频率衰减）
          2. 30天未出现 ×0.9（时间衰减）
          ↓
          频率 <0.3？
          ├─ 是 → 删除
          └─ 否 → 保留
          ↓
          数量 >500？
          ├─ 是 → 保留频率最高的 500 个
          └─ 否 → 全部保留
```

### 数学模拟

假设一个关键词初始频率 0.8，不再出现：

| 运行次数 | 频率衰减 | 时间衰减 | 最终频率 | 状态 |
|---------|---------|---------|---------|------|
| 0 | - | - | 0.80 | 加入 |
| 1 | ×0.95 | ×0.9 | 0.68 | 保留 |
| 2 | ×0.95 | ×0.9 | 0.58 | 保留 |
| 3 | ×0.95 | ×0.9 | 0.50 | 保留 |
| 4 | ×0.95 | ×0.9 | 0.43 | 保留 |
| 5 | ×0.95 | ×0.9 | 0.37 | 保留 |
| 6 | ×0.95 | ×0.9 | 0.31 | 保留 |
| 7 | ×0.95 | ×0.9 | 0.27 | 删除 ✓ |

**结论**：不再出现的关键词会在 7 次运行后被自动删除。

## 黑名单数量控制

### 理论上限

```
最多 500 个关键词（硬限制）
实际稳定在 50-150 个（自然平衡）
```

### 为什么不会无限增长？

1. **高频准入（>50%）**：只有频繁出现的关键词才加入
2. **频率衰减（×0.95）**：所有关键词每次衰减 5%
3. **时间衰减（×0.9）**：长期不出现额外衰减 10%
4. **低频清理（<0.3）**：低频关键词自动删除
5. **数量限制（500个）**：超过限制保留频率最高的

### 实际运行模拟

假设每天运行 3 次，每次有 5 条低分新闻：

```
第 1 天：
- 新增 10 个关键词（频率 >0.5）
- 黑名单：10 个

第 7 天：
- 新增 15 个关键词
- 旧关键词衰减，5 个被删除
- 黑名单：20 个

第 30 天：
- 新增 20 个关键词
- 旧关键词衰减，15 个被删除（时间衰减生效）
- 黑名单：25 个

第 90 天：
- 新增 10 个关键词
- 旧关键词衰减，20 个被删除
- 黑名单：15 个

稳定状态：
- 黑名单稳定在 50-150 个关键词
- 远低于 500 的上限
- 自动优化，无需手动维护
```

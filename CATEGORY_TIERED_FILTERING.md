# 分类分层过滤系统

## 设计理念

不同分类有不同的重要性层级，应该有不同的数量控制策略：

```
头条（最严格）
  ↓ 不够重要
政治/财经/科技（次级，相对宽松）
  ↓ 过于不重要
国际（兜底，有水位线）
  ↓ 完全不重要
丢弃 ❌
```

## 分层策略

### 智能水位线原则

不要一刀切地按比例削减，而是根据实际数量智能调整：

```python
if 数量 <= 低水位:
    保留全部  # 避免过度削减
elif 数量 <= 最大值:
    保留 max(数量 × 60%, 最大值 × 60%)  # 至少保留 60%
else:  # 数量 > 最大值
    保留 min(max(数量 × 60%, 最大值 × 80%), 最大值)  # 至少保留 80%，但不超过最大值
```

**设计理念**：
- 数量少的栏目（如科技）不会被过度削减
- 数量适中的栏目保留更多
- 数量超出的栏目也保留足够多，但不超过最大值
- 避免一刀切，根据实际情况智能调整

### 第一层：头条（最严格）

**水位线**：基于平方根缩放
```
1小时  → 低水位 10, 最大 20
8小时  → 低水位 28, 最大 56
24小时 → 低水位 49, 最大 98
```

**过滤策略**：
- 使用 LLM 评分，按重要性排序
- 保留最重要的 60%（不少于 8 条，不超过最大值）
- 不够重要的下放到次级分类（政治/财经/科技）

**示例**：
```
8小时报：
- 原始头条：80 条
- 低水位：28 条
- 保留：80 × 60% = 48 条（不超过最大值 56）
- 下放：32 条到次级分类
```

### 第二层：政治/财经/科技（次级，智能水位线）

**水位线**：头条的 1.5 倍
```
1小时  → 低水位 15, 最大 30
8小时  → 低水位 42, 最大 84
24小时 → 低水位 74, 最大 147
```

**智能过滤策略**：
- 接收从头条下放的新闻
- 按时间排序（最新的在前）
- 智能削减：
  - 数量 ≤ 42：全部保留
  - 42 < 数量 ≤ 84：保留 max(数量 × 60%, 50)
  - 数量 > 84：保留 min(max(数量 × 60%, 67), 84)
- 过于不重要的下放到国际

**示例对比**（8小时报）：

| 分类 | 原始 | 旧逻辑保留 | 新逻辑保留 | 差异 | 说明 |
|------|------|-----------|-----------|------|------|
| 科技 | 20 | 12 | 20 | +8 | 数量少，全部保留 ✓ |
| 科技 | 36 | 21 | 36 | +15 | 数量少，全部保留 ✓ |
| 政治 | 62 | 37 | 50 | +13 | 数量适中，至少保留 50 ✓ |
| 财经 | 90 | 54 | 67 | +13 | 数量超出，至少保留 67 ✓ |
| 政治 | 150 | 84 | 84 | 0 | 数量超出，不超过最大值 84 ✓ |

**优势**：
- 科技栏目（20-36条）不会被削减，全部保留
- 政治栏目（62条）保留更多（50 vs 37）
- 财经栏目（90条）保留更多（67 vs 54）
- 超大数量（150条）不会超过最大值（84）

### 第三层：国际（兜底，智能水位线）

**水位线**：与头条相同
```
1小时  → 低水位 10, 最大 20
8小时  → 低水位 28, 最大 56
24小时 → 低水位 49, 最大 98
```

**智能过滤策略**：
- 接收从次级分类下放的新闻
- 按时间排序（最新的在前）
- 智能削减：
  - 数量 ≤ 28：全部保留
  - 28 < 数量 ≤ 56：保留 max(数量 × 60%, 33)
  - 数量 > 56：保留 min(max(数量 × 60%, 44), 56)
- 完全不重要的直接丢弃 ❌

**示例**：
```
8小时报 - 国际分类：
- 原始分类：40 条
- 接收次级下放：50 条
- 总计：90 条
- 智能削减：保留 54 条（min(max(54, 44), 56) = 54）
- 丢弃：36 条 ❌
```

## 设计优势

### 1. 分层过滤，精准控制

- 头条：最严格，只保留最重要的新闻
- 次级：相对宽松，允许更多专业新闻
- 国际：兜底，但也有限制

### 2. 智能水位线，避免过度削减

- 数量少的栏目（如科技 20-36条）不会被削减，全部保留
- 数量适中的栏目保留更多（至少 60% 的最大值）
- 数量超出的栏目也保留足够多（至少 80% 的最大值）
- 避免一刀切，根据实际情况智能调整

### 3. 不丢失重要信息

- 从头条下放的新闻会重新分类到次级
- 从次级下放的新闻会进入国际
- 只有完全不重要的才会被丢弃

### 4. 自动平衡

- 每层都有水位线，自动控制数量
- 次级分类更宽松（1.5倍），允许更多专业内容
- 国际作为兜底，但也不会无限增长

### 5. 性能优化

- 头条使用 LLM 评分（最重要）
- 次级和国际使用时间排序（更快）
- 分层处理，避免对所有新闻都用 LLM

## 总结

**分层过滤 + 智能水位线的优势**：
1. ✅ 头条最严格，只保留最重要的
2. ✅ 次级相对宽松，允许更多专业内容
3. ✅ 国际兜底，但也有限制
4. ✅ 智能水位线，避免过度削减小栏目
5. ✅ 不丢失重要信息，逐层下放
6. ✅ 自动平衡，无需手动调整
7. ✅ 性能优化，只对头条用 LLM

**最终目标**：让每个分类都"恰到好处"，不多不少，小栏目不会被过度削减。
